const char* pfwGetLastError ( const PfwHandler *  handle)
Returns
a string describing the last call result. If the last pfw function call succeeded, return an empty string. If the last pfw function call failed, return a message explaining the error cause. The return pointer is invalidated if any pfw method is called on the SAME PfwHandle.

Each PfwHandle own it's last error message. It is not static nor TLS. As a result, calling a pfw function with a NULL PfwHandler will result in a failure WITHOUT updating the last error.

bool pfwSetCriterion ( PfwHandler *  handle,
const char  name[],
int  value 
)

Set a criterion value given its name and value.

Parameters
handle[in]
See also
PfwHandler
Parameters
name[in]The name of the criterion that need to be changed.
valueIf the criterion is exclusive, the index of the new value. If the criterion is inclusive, a bit field where each bit correspond to the value index. For an inclusive criterion defined as such: { "Red", "Green", "Blue", NULL } to set the value Green and Blue, value has to be 1<<1 | 1<<2 = 0b110 = 6. For an exclusive criterion defined as such: { "Car", "Boat", "Plane", NULL } to set the value Plane, value has to be 2.

Criterion change do not have impact on the parameters value (no configuration applied) until the changes are committed using pfwApplyConfigurations.

Returns
true on success and false on failure.
